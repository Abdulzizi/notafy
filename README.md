# Notafy — OCR Nota & Struk Otomatis

> **Ubah foto nota jadi data terstruktur dalam detik.**

Notafy is a Laravel-based expense receipt OCR tool built for Indonesian companies.
It extracts structured data from employee receipts — digital PDFs, app screenshots,
and thermal struk photos — and outputs audit-ready JSON for reimbursement workflows.

Finance and audit teams upload receipts from platforms like Shopee, GoFood, Grab, and
Indomaret. Notafy reads them automatically, flags low-confidence results for manual review,
and stores a full extraction history per user.

## Supported Platforms

| Platform | Type | Confidence |
|----------|------|------------|
| Shopee | E-commerce PDF / screenshot | ~95% |
| Tokopedia | E-commerce PDF / screenshot | ~95% |
| GoFood | App screenshot / PDF | ~92% |
| GoCar | App screenshot | ~91% |
| GoRide | App screenshot | ~90% |
| GrabFood | App screenshot / PDF | ~90% |
| GrabBike | App screenshot | ~89% |
| GrabCar | App screenshot | ~88% |
| Indomaret | Thermal struk scan | ~75% |
| Nota warung | Handwritten / mixed print | 45–60% |

> Confidence rates reflect average `confidence_score` across real samples. Results below
> **0.75** are automatically flagged `needs_review = true`.

## Tech Stack

| Layer | Technology |
|-------|------------|
| Framework | Laravel 12 (PHP 8.2+) |
| OCR engine (default) | Tesseract OCR — local, free, no API calls |
| OCR engine (accurate) | Mistral OCR API (`mistral-ocr-latest`) |
| Structured extraction | Mistral Chat API (`mistral-small-latest`) |
| Database | MySQL / SQLite (SQLite for local dev) |
| Auth | Laravel + Google SSO (Socialite) |
| Billing | Stripe (Cashier) + Mayar (IDR gateway) |
| File processing | Spatie PDF-to-Image + ImageMagick + Intervention Image |

A GoFood receipt photo processed through Mistral OCR returns:

```json
{

  "category": "food",
  "transaction_id": "F-20240315-9871234",
  "transaction_date": "2024-03-15",
  "transaction_time": "12:34",
  "vendor_name": "Warung Makan Bu Tuti",
  "employee_name": "Budi Santoso",
  "subtotal": 42000,
  "discount": 5000,
  "delivery_fee": 9000,
  "service_fee": 2000,
  "tax": null,
  "total_amount": 48000,
  "payment_method": "GoPay",
  "source_type": "digital_jpg",
  "confidence_score": 0.92,
  "needs_review": false,
  "raw_text": "Thanks for using GoFood\nTransaction ID: F-20240315-9871234\nWarung Makan Bu Tuti\n..."
```

`raw_text` always contains the full unprocessed OCR output for audit trail purposes.

## Setup

### Requirements

- PHP 8.2+
- Composer
- Node.js + npm
- Tesseract OCR installed locally (`apt install tesseract-ocr tesseract-ocr-ind`)
- ImageMagick (`apt install imagemagick`)
- MySQL or SQLite

### Install

```bash
git clone https://github.com/your-org/notafy.git
cd notafy

composer install
npm install

cp .env.example .env
php artisan key:generate
```

### Configure `.env`

See the [Environment Variables](#environment-variables) section below, then:

```bash
php artisan migrate
npm run build
php artisan serve
```

For local development with queue + logs:

```bash
composer run dev
```

This starts `php artisan serve`, `queue:listen`, `pail`, and `npm run dev` concurrently.

## OCR Schema

Every receipt processed through Mistral OCR produces the following structured output,
stored directly in the `ocr_results` table:

| Field | Type | Description |
|-------|------|-------------|
| `platform` | string | `shopee` \| `tokopedia` \| `gofood` \| `gocar` \| `goride` \| `grabfood` \| `grabbike` \| `grabcar` \| `indomaret` \| `nota_warung` \| `unknown` |
| `category` | string | `transport` \| `food` \| `belanja` \| `other` |
| `transaction_id` | string | Order ID / No. Pesanan / Booking Code |
| `transaction_date` | date | `YYYY-MM-DD` |
| `transaction_time` | string | `HH:mm` |
| `vendor_name` | string | Restaurant, store, or seller name |
| `employee_name` | string | Passenger name from transport receipts |
| `subtotal` | integer | IDR, before discounts and fees |
| `discount` | integer | IDR discount applied |
| `delivery_fee` | integer | IDR delivery/ongkir charge |
| `service_fee` | integer | IDR platform service fee |
| `tax` | integer | PPN in IDR |
| `total_amount` | integer | **Always present.** Final amount paid, IDR. |
| `payment_method` | string | `GoPay` \| `LinkAja` \| `OVO` \| `Bank BRI` \| `COD` \| etc. |
| `source_type` | string | `digital_pdf` \| `digital_jpg` \| `thermal_scan` \| `photo_hardcopy` |
| `confidence_score` | float | `0.0–1.0`. Notafy's receipt readability assessment. |
| `needs_review` | boolean | `true` when `confidence_score < 0.75` |
| `extracted_text` | text | Full raw OCR output — always preserved for audit trail |

> Tesseract-only results populate `extracted_text` and `confidence` (Tesseract's
> own 0–100 word-level score) but leave all schema fields `null`. Run Mistral rerun
> (`POST /ocr/result/{id}/rerun`) to get the full structured extraction.
